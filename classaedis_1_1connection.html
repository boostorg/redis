template<class AsyncReadWriteStream = boost::asio::ip::tcp::socket>
class aedis::connection< AsyncReadWriteStream >

A high level Redis connection class.

This class keeps a healthy connection to the Redis instance where commands can be sent at any time. For more details, please see the documentation of each individual function.

Starts communication with the Redis server asynchronously.

This function performs the following steps

  • Resolves the Redis host as of async_resolve with the timeout passed in connection::config::resolve_timeout.
  • Connects to one of the endpoints returned by the resolve operation with the timeout passed in connection::config::connect_timeout.
  • Starts the idle check operation with the timeout of twice the value of connection::config::ping_interval. If no data is received during that time interval async_run completes with error::idle_timeout.
  • Starts the healthy check operation that sends command::ping to Redis with a frequency equal to connection::config::ping_interval.
  • Starts reading from the socket and delivering events to the request started with async_exec or async_read_push.

For an example see echo_server.cpp.

Parameters
hostRedis address.
portRedis port.
tokenCompletion token.

The completion token must have the following signature

void f(boost::system::error_code);
Returns
This function returns only when there is an error.

void aedis::connection< AsyncReadWriteStream >::cancel_run ( )
inline

Closes the connection with the database.

Calling this function will cause async_run to return. It is safe to try a reconnect after that, when that happens, all pending request will be automatically sent.

Note however that the prefered way to close a connection is to send a quit command if you are actively closing it. Otherwise an unresponsive Redis server will cause the idle-checks to kick in and lead to async_run returning with idle_timeout.
