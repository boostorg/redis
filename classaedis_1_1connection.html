Detailed Description

template<class AsyncReadWriteStream = boost::asio::ip::tcp::socket>
class aedis::connection< AsyncReadWriteStream >

A high level connection to Redis.

This class keeps a healthy connection to the Redis instance where commands can be sent at any time. For more details, please see the documentation of each individual function.

Remarks
This class exposes only asynchronous member functions, synchronous communications with the Redis server is provided by the sync class.
Template Parameters
AsyncReadWriteStreamA stream that supports async_read_some and async_write_some.

template<class AsyncReadWriteStream = boost::asio::ip::tcp::socket>
template<class CompletionToken = boost::asio::default_completion_token_t<executor_type>>
auto aedis::connection< AsyncReadWriteStream >::async_run ( CompletionToken  token = CompletionToken{})
inline

Starts communication with the Redis server asynchronously.

This function performs the following steps

  • Resolves the Redis host as of async_resolve with the timeout passed in config::resolve_timeout.
  • Connects to one of the endpoints returned by the resolve operation with the timeout passed in config::connect_timeout.
  • Starts healthy checks with a timeout twice the value of config::ping_interval. If no data is received during that time interval connection::async_run completes with error::idle_timeout.
  • Starts the healthy check operation that sends PINGs to Redis with a frequency equal to config::ping_interval.
  • Starts reading from the socket and executes all requests that have been started prior to this function call.
Remarks
When a timeout occur and config::enable_reconnect is set, this function will automatically try a reconnection without returning control to the user.

For an example see echo_server.cpp.

Parameters
tokenCompletion token.

The completion token must have the following signature

void f(boost::system::error_code);

template<class AsyncReadWriteStream = boost::asio::ip::tcp::socket>
std::size_t aedis::connection< AsyncReadWriteStream >::cancel ( operation  op)
inline

Cancel operations.

  • operation::exec: Cancels operations started with async_exec.
  • operation::run: Cancels async_run. Notice that the preferred way to close a connection is to ensure config::enable_reconnect is set to false and send QUIT to the server. An unresponsive Redis server will also cause the idle-checks to kick in and lead to connection::async_run completing with error::idle_timeout. Calling cancel(operation::run) directly should be seen as the last option.
  • operation::receive_event: Cancels connection::async_receive_event.
Parameters
opThe operation to be cancelled.
Returns
The number of operations that have been canceled.
