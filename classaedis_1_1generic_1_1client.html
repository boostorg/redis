Detailed Description

template<class AsyncReadWriteStream, class Command>
class aedis::generic::client< AsyncReadWriteStream, Command >

A high level Redis client.

This class represents a connection to the Redis server. Some of its most important features are

  1. Automatic management of commands. The implementation will send commands and read responses automatically for the user.
  2. Memory reuse. Dynamic memory allocations will decrease with time.

For more details, please see the documentation of each individual function.

Starts communication with the Redis server asynchronously.

This class performs the following steps

  • Resolve the Redis host as of async_resolve with the timeout passed in client::config::resolve_timeout.
  • Connect to one of the endpoints returned by the resolve operation with the timeout passed in client::config::connect_timeout.
  • Start the async read operation that keeps reading incoming responses. Each individual read uses the timeout passed on client::config::read_timeout. After each successful read it will call the read or push callback.
  • Start the async write operation that wait for new commands to be sent to Redis. Each individual write uses the timeout passed on client::config::write_timeout. After a successful write it will call the write callback.
  • Start the check idle operation with the timeout specified in client::config::idle_timeout. If no data is received during that time interval async_run returns generic::error::idle_timeout.
  • Start the healthy check operation that sends redis::command::ping to Redis with a frequency equal to client::config::idle_timeout / 2.

In addition to the callbacks mentioned above, the read operations will call the resp3 callback as soon a new chunks of data become available to the user.

Parameters
hostIp address or name of the Redis server.
portPort where the Redis server is listening.
tokenThe completion token.

The completion token must have the following signature

void f(boost::system::error_code);

Notice this function returns only when there is an error.

Remarks
  • It is safe to call this function again after it has returned. In that case, any outstanding commands will be sent and not get lost.
  • If a disconnect occurs while the response to a request has not been received, the client doesn't try to resend it to avoid resubmission problem.
